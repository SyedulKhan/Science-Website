Polymorphism
Definition

Where one object can be a part of more than one category of objects.

Visualising polymorphism

This would be a basic image of what polymorphism is and how it works. This means that a variable that stores a Vehicle object, can store any descendant of the Vehicle class.

Types of Polymorphism

Polymorphism can be one of 2 types, in this case it can be either static or dynamic.

  • Static Polymorphism - The response to a function is determined at the compile time.
  • Dynamic Polymorphism - It is decided at run-time.
What is Static Polymorphism?

Static polymorphism is basically the apparatus of linking a function with an object during compile time. This is known as either early binding or static binding.

C# provides two techniques to implement static polymorphism. Which are known as:

  • Function overloading
  • Operator overloading
Function overloading

This is where the user can have a multiple amount of definitions for an identical function name which belongs in the same scope. The definition of the function however must be different from each other by the types. The user cannot surplus function declarations that differ only by return type.

Operator overloading

Operator overloading is where the user can overload or redefine most of the built in available in c#. Overloaded operators are basically functions which contain special names followed by a keyword along with a symbol for the operator which is defined. Furthermore, it can also be stated that an overloaded operator has a return type along with a parameter list.

What is Dynamic Polymorphism?

Dynamic Polymorphism allows the user to create abstract classes which are used to provide partial class implementation of an interface. Initially the implementation is complete when a derived class in inherits from it. These abstract classes contain abstract methods which have been implemented by the derived classes which have more of a specialised functionality.

Putting it in practise with examples
  • Vehicle v0 = new Vehicle();
  • Vehicle v1 = new Tank();
  • Vehicle v2 = new GolfMk2();

This is the ability in which objects of different types provide a unique interface for different implementations of methods. Usually, used within the context of late binding where the behavior of an object replies to a call to its method members is determined based on object type at run time. Polymorphism permits redefining methods in derived classes. Polymorphism forms one of the essential perceptions of object-oriented programming, along with inheritance and encapsulation.
